# 课间枪战 · 第一人称多人 FFA 枪战

一个**纯静态**的第一人称多人 FFA（自由混战）枪战网页游戏，给中学生课间玩。
同学之间靠 **填一样的房间号** 联机。

- 无服务端、无构建步骤，所有路径都是相对路径。
- 直接传到 **Vercel / GitHub Pages** 就能跑。
- 第三方库已经 **vendor 到本地 `/vendor`**，用普通 `<script src>` 引入，
  **运行时零外部 CDN 依赖**（绝不使用 importmap / esm.sh —— 上个版本就是栽在这）。

## 本地运行

随便起一个静态服务器即可（**不要**用 `file://` 直接打开，WebRTC/指针锁定在 http 下更稳）：

```bash
# 方式一：Python
python -m http.server 8000
# 然后浏览器打开 http://localhost:8000

# 方式二：Node
npx serve .
```

> **瞄准/鼠标锁不住？** 第一人称瞄准依赖浏览器的"指针锁定"。请在**真正的浏览器标签页**
> 打开 `http://localhost:8000/`，进入战场后**点一下画面**即可锁定鼠标开始瞄准（按 `Esc` 松开）。
> 编辑器/IDE 里的**内嵌预览**通常会禁用指针锁定（iframe 限制），在那里会瞄不动——这不是游戏的问题，
> 换成真正的浏览器标签页就正常了。

## 部署

### Vercel
1. 把整个文件夹（至少 `index.html` + `vendor/`）传上去 / 连 Git 仓库。
2. Framework Preset 选 **Other**，**不需要** Build Command、不需要 Output 设置。
3. 部署完成，访问分配的地址即可。

### GitHub Pages
1. 把文件推到仓库，Settings → Pages → 选分支根目录。
2. 等几分钟，访问 `https://<用户名>.github.io/<仓库名>/`。

> 因为全是相对路径，放在子路径（如 `/<仓库名>/`）下也能正常加载。

## 联机模型（怎么"填同一个房间号就在一起"）

- 进游戏前填：**名字 + 房间号**。填一样房间号的人进同一局。
- 用房间号 **确定性地** 生成一个主机 peer id：`ffaclash-<房间号>-host`。
- **第一个占用该 id 的人当主机**；其余人发现 id 被占用，就作为客户端连到主机。
  采用 **星型拓扑**：主机转发其他人的状态（简单可靠）。
- 每个客户端约 **18Hz** 广播自己的状态（位置 / 朝向 / 当前武器 / 血量 / 存活）。
- 命中由 **开枪方上报给被击中方**；被击中方扣自己的血，死亡后广播 `kill` 事件更新计分。
- 正确处理 **加入 / 离开**（增删对应的小人和计分条）；并做了基础的 **主机迁移**：
  主机掉线时，客户端会尝试重新占用主机 id 接管房间。

### WebRTC / NAT 说明
PeerJS 默认用它的 **公共 broker** 做信令（这部分需要联网，属正常），并已配 Google/Twilio 的 STUN。
WebRTC 受 NAT 穿透限制：**在个别对称型 NAT / 严格防火墙下两端可能连不上**。若遇到连不上：
- 可改为 **自建 PeerServer** 做信令（`npm i -g peer && peerjs --port 9000`），
  在 `index.html` 的 `PEER_OPTS` 里增加 `{ host, port, path }` 指向它；
- 并/或在 `iceServers` 里加一个 **TURN** 服务器做中继兜底。

当前版本用公共 broker 即可满足课间联机需求。

## 手动联机自测（两个标签页）

1. 本地起静态服务器（见上），或用已部署的地址。
2. 开 **两个浏览器标签页**（或两台电脑），都打开 **同一个地址**。
3. 两边填 **同一个房间号**（名字随意取不同的，方便区分）。
4. 各自点"进入战场"，点一下画面锁定鼠标。
5. 确认：
   - 两边都能在场上看到 **对方的小人**（带名字标签 + 头顶血条）。
   - 互相移动时，对方的位置是 **平滑** 的（有插值）。
   - 能 **互相打到**：开枪命中对方时出现命中标记，对方掉血；
     打死对方时右上角出现 **击杀提示**，积分 +1。
   - 关掉一个标签页，另一边对应的小人和计分条会 **消失**。

> 提示：同一台电脑开两个**普通**标签页即可；用一个普通 + 一个无痕窗口也行。
> 第一个进房间的人是主机，第二个及以后是客户端。

## 自测（headless 测试）

三个测试都会启一个内置静态服务器，用 headless Chromium 加载页面跑断言。

```bash
npm install                       # 安装 dev 依赖 playwright（部署不需要）
npx playwright install chromium
npm test                          # 依次跑下面三个测试
# 或分开跑：
npm run smoke                     # = node test/smoke.mjs    冒烟（加载/库/canvas/booted）
npm run gameplay                  # = node test/gameplay.mjs 单机玩法 + 联机逻辑
npm run feel                      # = node test/feel.mjs     手感：后坐/换弹动画/走路摆动/枪口闪光
```

**`smoke.mjs` —— 冒烟测试（对应验收第 2 条）**，断言：
控制台零报错 / `window.THREE` 与 `window.Peer` 已定义 / canvas 存在 / `window.__booted === true`。
它不点击"进入战场"，所以不触发 PeerJS 联网，控制台保持干净。

**`gameplay.mjs` —— 单机玩法 + 联机逻辑测试**，在 `?test=1` 离线模式下真实驱动游戏：
按 W 能前进、换枪生效、开火消耗弹药并生成弹道、换弹装满、受伤扣血；
再喂合成消息进真实的消息处理函数，验证"对端加入/离开增删小人、收到命中扣血、击杀计分"——
这部分不依赖 WebRTC 传输（传输层由上面的"两个标签页"手动测试覆盖）。

> `?test=1` 是测试专用：进入离线模式并暴露 `window.__test` 钩子。
> 正常游玩**不要**带这个参数。

## 实现要点（给想改的人）

- **Three.js r137 UMD**：普通 `<script>` 引入即得全局 `THREE`，不用任何 ESM/CDN。
- **第一人称控制全手写**：指针锁定 + `mousemove` 累加 yaw/pitch + 逐轴 AABB 碰撞，
  不依赖 `PointerLockControls`（全局构建里没有它）。进入战场后只要鼠标没锁定，
  **点击画面任意处即可（重新）锁定鼠标开始瞄准**，按 `Esc` 松开。
- **手感动画**（`updateViewBobAndGun`）：走路时相机有头部摆动（只偏移位置、不改朝向，
  所以**不影响瞄准准星**）；武器有跟手摇晃、待机呼吸、开火后坐（后顶+上抬+枪口星芒闪光）、
  换弹时枪械下沉前倾的换弹动画。全部尊重 `prefers-reduced-motion`（开启时关闭这些晃动）。
- **库存在性检查**：`index.html` 在引入库后立刻检查 `window.THREE` / `window.Peer`，
  缺哪个就在页面顶部红条里 **明确报出是哪一个库**。
- **音效**：全部用 WebAudio 现场合成（开枪 / 命中 / 死亡 / 换弹），无任何音频素材文件。
- **可访问性**：尊重 `prefers-reduced-motion`（关闭镜头抖动/菜单环绕）、输入有键盘可见焦点、
  窗口 `resize` 自适应。
- **引擎与联机解耦**：3D 引擎在打开页面时就启动（菜单背景里能看到竞技场），
  只有点"进入战场"后才连 PeerJS——所以 headless 自测无需联网即可通过。
